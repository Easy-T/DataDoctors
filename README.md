# DataDoctors
To achieve SQLP, Go DataDoctors :)

# 실습 코드리뷰 진행 순서
1) Pull Request 방식 사용
2) 각자 작업을 깃허브에 반영할 수 있도록, "SQLP_튜닝_이름" 형식의 브랜치로 네이밍 할 것.
3) 로컬 브랜치 작업 <-> 깃허브 브랜치 작업에 해당하는 브랜치는 오직 자신의 "SQLP_튜닝_이름"일 것.
4) 남의 깃허브 브랜치는 로컬로 채크아웃 후, 오직 코드리뷰 목적의 Pull Request로만 사용할 것.
5) 따라서 자신에게 온 Pull Request는 스스로 merge할 것.(중요)
